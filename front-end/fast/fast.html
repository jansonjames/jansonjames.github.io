# 快速开发 beta 1.0

# 背景

工作多年，开发过不少项目，开发过程中有一些需求是通用并且出现频率极高的，每次都从0开始一行一行地写代码的方式无疑是低效的，那么如何提高效率，缩短开发时间呢？

# 提升效率的方式

  • 利用模块化和组件化开发：将项目拆分成独立的模块和组件，利用模块化和组件化的开发方式可以提高代码的可维护性和复用性。
  • 组件库：将一些通用的UI组件、布局组件或功能组件提取出来，创建组件库。这样就可以在项目中重复使用这些组件，减少重复劳动并保持一致的用户体验。
  • 脚手架：使用脚手架工具，如Vue CLI来自动化项目的初始化和配置过程。脚手架可以快速搭建项目结构，并提供一些常用的开发工具和依赖，减少重复性工作。
  • 使用第三方库：利用现有的第三方库可以快速解决一些常见的开发需求，如日期处理、表单验证、数据可视化等。
  • 借助AI生成代码：目前市场上有一些自动补全代码的AI工具，比如微软的GitHub copilot等。
  • 自定义代码片段：通过对过往项目进行复盘，然后总结出通用的代码记录在文档中，需要使用的时候直接复制黏贴，或者一种更快的方式是在编辑器中预设好代码片段，然后开发的时候只要输入关键字就可以生成代码，加速开发。

# 根据设计图定制化场景

C端项目基本上对UI的要求都是很高的，在使用第三方UI框架和自定义组件的过程中都发现了这样的问题：因为样式差别很大，哪怕是几个像素的差别，圆角直角的差异等等，都需要调整css样式，另外，有的时候某些页面结构相似，但又有些许差别，原先写好的组件用起来，需要调整，把差别都调整完融入到一个组件里，随着开发数量的增加，组件越来越大，越来越复杂，也变得不太好使用和维护，多种原因导致使用组件的过程中不那么流畅，于是选择了自定义代码片段的方式，在编辑器中预设好代码片段，开发过程中只要输入关键字就可以生成通用代码，然后在此基础上编写样式和逻辑代码，从而提升效率。

# 最后
